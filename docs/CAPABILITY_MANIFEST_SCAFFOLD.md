# Capability Manifest Scaffold

Last-Updated: 2026-06-24

This sprint adds the manifest and registry scaffold only.

- No new runtime dependencies are added.
- No P0 tools are integrated yet.
- The explicit-file sidecar subsystem remains closed and unchanged.
- Capability manifests are the future guardrail for optional tools.
- Heavy CPU/memory work and external binaries must be explicitly approved before execution.
- Future UI work should read manifests to generate settings, command palette entries, approval cards, and capability warnings.

This document is intentionally short. It is a pointer for future capability work, not a runtime design spec.
